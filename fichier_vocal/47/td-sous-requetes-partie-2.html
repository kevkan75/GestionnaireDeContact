Partie 1: Sous-requêtes de lignes (Row Subqueries)

Introduction aux sous-requêtes de lignes

Une sous-requête de ligne retourne une ligne unique pouvant contenir plusieurs colonnes, contrairement aux sous-requêtes scalaires qui retournent une seule valeur.

Opérateurs disponibles pour les comparaisons de lignes:

=  >  <  >=  <=  <>  !=  <=>

Exercice 1.1: Comparaison de lignes simples

Voici des insertions SQL à exécuter:

-- Insérer le produit Smartphone XS
INSERT INTO products (name, description, price, stock, category, supplier_id, status)
VALUES ('Smartphone XS', 'Smartphone haut de gamme compact', 599.99, 25, 'Electronics', 2, 'Active');

-- Insérer d'autres produits avec le même prix et stock que Smartphone XS
INSERT INTO products (name, description, price, stock, category, supplier_id, status)
VALUES 
('Écouteurs Pro', 'Écouteurs sans fil premium', 599.99, 25, 'Audio', 4, 'Active'),
('Montre Connectée X5', 'Montre intelligente avec suivi de santé', 599.99, 25, 'Wearables', 5, 'Active');

Trouvez tous les produits dont le prix et le stock correspondent exactement à ceux du produit "Smartphone XS" (utilisez une sous-requête de ligne).

Solution

SELECT 
    product_id,
    name,
    price,
    stock
FROM 
    products
WHERE 
    (price, stock) = (
        SELECT price, stock 
        FROM products 
        WHERE name = 'Smartphone XS'
    );

Exercice 1.2: Constructeur de lignes

Identifiez tous les produits pour lesquels la combinaison de prix et de stock est strictement supérieure à celle du produit ayant l'ID 5 (utilisez ROW pour la comparaison).

Solution

SELECT 
    product_id,
    name,
    price,
    stock
FROM 
    products
WHERE 
   ROW(price, stock) > (
        SELECT price, stock 
        FROM products 
        WHERE product_id = 5
    );
    
-- Notez que, pour ce type de requêtes, écrire ROW est optionnel.

Exercice 1.3: Comparaison multiple de lignes avec IN

Trouvez tous les produits dont la combinaison de catégorie et de supplier_id existe également pour des produits ayant un prix supérieur à 500.

Solution

SELECT 
    product_id,
    name,
    category,
    supplier_id,
    price
FROM 
    products
WHERE 
    ROW(category, supplier_id) IN (
        SELECT category, supplier_id
        FROM products
        WHERE price > 500
    )
AND price >= 500;

Exercice 1.4: Comparaison NOT IN avec lignes

Identifiez tous les clients dont aucune combinaison (loyalty_level, active) n'existe pour les clients ayant un achat total supérieur à 1000.

Solution

SELECT 
    c.customer_id,
    c.name,
    c.loyalty_level,
    c.active
FROM 
    customers c
WHERE 
    ROW(c.loyalty_level, c.active) NOT IN (
        SELECT c2.loyalty_level, c2.active
        FROM customers c2
        JOIN orders o ON c2.customer_id = o.customer_id
        GROUP BY c2.customer_id, c2.loyalty_level, c2.active
        HAVING SUM(o.total_amount) > 1000
    );

Partie 2: Tables dérivées

Introduction aux tables dérivées

Une table dérivée est une sous-requête dans la clause FROM qui génère une table temporaire utilisable dans la requête principale. Elle doit toujours avoir un alias et peut contenir plusieurs colonnes.

Exercice 2.1: Table dérivée simple

Créez une requête qui utilise une table dérivée pour calculer le prix moyen par catégorie, puis affichez uniquement les catégories dont le prix moyen est supérieur à 200.

Solution

SELECT 
    category_avg.category,
    category_avg.avg_price
FROM (
    SELECT 
        category,
        AVG(price) AS avg_price
    FROM 
        products
    GROUP BY 
        category
) AS category_avg
WHERE 
    category_avg.avg_price > 200
ORDER BY 
    category_avg.avg_price DESC;

Exercice 2.2: Table dérivée avec noms de colonnes explicites

Créez une table dérivée qui calcule pour chaque catégorie: le nombre de produits, le prix minimum, le prix maximum et le prix moyen. Nommez explicitement les colonnes de la table dérivée.

Solution

SELECT 
    cat_stats.category_name,
    cat_stats.product_count,
    cat_stats.min_price,
    cat_stats.max_price,
    cat_stats.avg_price
FROM (
    SELECT 
        category AS category_name,
        COUNT(*) AS product_count,
        MIN(price) AS min_price,
        MAX(price) AS max_price,
        AVG(price) AS avg_price
    FROM 
        products
    GROUP BY 
        category
) AS cat_stats (category_name, product_count, min_price, max_price, avg_price)
ORDER BY 
    cat_stats.product_count DESC;

Exercice 2.3: Tables dérivées pour agrégations imbriquées

Pour chaque client, calculez l'écart entre le montant de sa commande la plus importante et la moyenne des montants de toutes ses commandes.

Solution

SELECT 
    c.customer_id,
    c.name,
    order_stats.max_amount,
    order_stats.avg_amount,
    (order_stats.max_amount - order_stats.avg_amount) AS amount_difference
FROM 
    customers c
JOIN (
    SELECT 
        customer_id,
        MAX(total_amount) AS max_amount,
        AVG(total_amount) AS avg_amount
    FROM 
        orders
    GROUP BY 
        customer_id
) AS order_stats ON c.customer_id = order_stats.customer_id
ORDER BY 
    amount_difference DESC;

Exercice 2.4: Tables dérivées multiples

Créez une requête avec deux tables dérivées: une pour calculer les statistiques de ventes par catégorie et une autre pour calculer les statistiques d'évaluations par catégorie. Joignez-les pour analyser la corrélation entre les ventes et les évaluations.

Solution

SELECT 
    sales_stats.category,
    sales_stats.total_sales,
    sales_stats.items_sold,
    review_stats.avg_rating,
    review_stats.review_count
FROM (
    SELECT 
        p.category,
        SUM(oi.quantity * oi.price_at_time) AS total_sales,
        SUM(oi.quantity) AS items_sold
    FROM 
        order_items oi
    JOIN 
        products p ON oi.product_id = p.product_id
    GROUP BY 
        p.category
) AS sales_stats
JOIN (
    SELECT 
        p.category,
        AVG(pr.rating) AS avg_rating,
        COUNT(pr.review_id) AS review_count
    FROM 
        product_reviews pr
    JOIN 
        products p ON pr.product_id = p.product_id
    GROUP BY 
        p.category
) AS review_stats ON sales_stats.category = review_stats.category
ORDER BY 
    sales_stats.total_sales DESC;

Exercice 2.5: Fenêtrage avec tables dérivées

À l'aide d'une table dérivée, classez les clients par montant total de leurs achats au sein de leur niveau de fidélité (loyalty_level) et affichez leur rang. Utilizé RANK OVER.

Solution

SELECT 
    ranked_customers.customer_id,
    ranked_customers.name,
    ranked_customers.loyalty_level,
    ranked_customers.total_spent,
    ranked_customers.spending_rank
FROM (
    SELECT 
        c.customer_id,
        c.name,
        c.loyalty_level,
        SUM(o.total_amount) AS total_spent,
        RANK() OVER (
            PARTITION BY c.loyalty_level 
            ORDER BY SUM(o.total_amount) DESC
        ) AS spending_rank
    FROM 
        customers c
    JOIN 
        orders o ON c.customer_id = o.customer_id
    GROUP BY 
        c.customer_id, c.name, c.loyalty_level
) AS ranked_customers
ORDER BY 
    ranked_customers.loyalty_level,
    ranked_customers.spending_rank;
